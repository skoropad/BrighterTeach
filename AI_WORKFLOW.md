**ROLE:**
You are an Expert AI-Native Fullstack Engineer. We are building an MVP for a kids' EdTech product called "Brighterly AI Homework Helper" in a strict 2-3 hour timebox. Optimize for speed, functional completeness, and clean code. Do not over-engineer.

* Next.js (App Router), React, Tailwind CSS, Vercel AI SDK (`@ai-sdk/anthropic`)

**PHASE 1: AUDIT & PLAN (DO THIS FIRST AND PAUSE)**
Review the entire Next.js project structure generated by v0. Point out any hardcoded mock data, messy SVGs, or logic gaps. Then, generate a step-by-step plan outlining how you will implement the state management across the components, the chat locking UI, and the dynamic API route.
**STOP after generating the plan.**

*(For your reference, here is what the plan must accomplish once approved):*

**1. State Management & UI Logic (Frontend Components):**

* Implement state for `grade`, `subject`, `mathProblem`, `readingText`, and `readingQuestion` in the appropriate client components.

* Integrate `useChat` from Vercel AI SDK into the main chat interface.

* **Chat Lock Logic:** Disable the chat input in the Right Column UNTIL the user submits their first task via the Left Column buttons.

* **Payload:** Combine text inputs logically, attach `grade`, `subject`, and `mode` to the request, and trigger `append` or `handleSubmit`.

* **Clear History:** Wire the "Start New Session" button to wipe messages, inputs, and `localStorage`.

**2. Backend Logic (`app/api/chat/route.ts`):**

* Extract `messages`, `grade`, `subject`, and `mode` from `req.json()`.

* **CRITICAL - Dynamic System Prompt:**

  * *Subject Math:* Format equations clearly. Break down steps.

  * *Subject Reading:* Focus on comprehension and context.

  * *Mode Explain:* Complete, step-by-step explanation.

  * *Mode Hint:* STRICT RULE. Do NOT solve the problem. ONE small conceptual nudge.

**3. Bonus:** Persist `messages` array to `localStorage`.
